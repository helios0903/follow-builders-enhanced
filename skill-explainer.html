Claude Code Skill

What is a Skill?

A reusable instruction set that teaches Claude how to do a specific job — and the story of how we upgraded the Follow Builders digest.

Concept

A skill is Claude's instruction manual

Three components, one job: tell Claude when to act, how to act, and give it the tools it needs.

🧠

SKILL.md

SKILL.md

The brain. A markdown file that defines when to trigger (via the description field) and what to do (step-by-step instructions). Claude reads this every time the skill activates.

⚙️

Scripts

scripts/

The hands. Node.js or shell scripts that handle deterministic work — fetching data, sending emails, formatting files. No AI needed for these, just reliable execution.

✍️

Prompts

prompts/

The style guide. Markdown files that define how Claude should write — tone, format, what to include or skip. Loaded into context when the skill runs.

What changed

Three problems, three fixes

Each fix required a different layer of the skill stack.

✅ Fixed

1. Email formatting

Plain text. No sections, no hierarchy, everything looks the same.
↓
HTML email with headings, sections, inline links, and mobile-friendly layout via deliver-html.js.
✅ Fixed

2. Missing tweet links

Only the first tweet URL included per builder. Context lost.
↓
Every substantive tweet gets its own hyperlink inline in the summary, via explicit rules in remix-prompt.md.
✅ Fixed

3. No Chinese translation

Cron bypassed Claude entirely. Raw JSON → plain text. No AI = no translation.
↓
run-digest.sh pipes data through claude -p headless mode, giving every scheduled run full AI remix + bilingual output.
